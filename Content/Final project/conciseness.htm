Conciseness

Conciseness in writing UI messages means delivering essential information efficiently and effectively. Users interact with interfaces to complete tasks quickly, so UI messages need to be short, clear, and easy to scan. Additionally, technical constraints, like strict character limits, often require writers to be inventive with language and grammar while still maintaining clarity and usability.

Why conciseness matters

Concise UI messages help to:

  • Enhance usability – short, clear messages help users navigate interfaces with minimal effort.

  • Reduce cognitive load – less text means less mental processing, making it easier for users to absorb key information.

  • Speed up decision-making – users can quickly understand what to do next without unnecessary distractions.

  • Improve accessibility – concise text benefits users with cognitive disabilities or those reading on small screens.

How to write concisely

Concise UX writing leads to a smoother, faster, and more intuitive user experience. Key recommendations for concise UI messages are:

  • Use as few words as possible. Avoid unnecessary and redundant information.

  • Focus on user goals and make sure that you create content for an actual use case.

  • Avoid long, complex sentences.

  • Remove filler words (“basically,” “very,” “absolutely”).

  • Avoid redundancy (avoid “help and support”, use “help” or “support” instead).
